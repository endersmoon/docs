# Button




 
### Overview

Buttons are used to trigger actions. They can be used to submit forms, trigger modal dialogs, or perform other actions.





### Variants

We have 5 variants of buttons:

- `Primary`
- `outline`
- `ghost`
- `link`
- `destructive`

#### Primary

Use a default button for the main call to action on a page.
Avoid using multiple default buttons on a single page. 
Having more than one main call to action reduces their impact, and makes it harder for users to know what to do next.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button className=" bg-blue-600">Button</Button>
</div>

#### Outline
Use Outline buttons for Outline calls to action on a page.Pages with too many calls to action make it hard for users to know what to do next. Before adding lots of Outline buttons, try to simplify the page or break the content down across multiple pages.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button variant='outline' className=' border-gray-200'>
    Button
  </Button>
</div>

#### Ghost

Ghost button are a great alternative to outline buttons when you need to make a button less prominent.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button variant='ghost'>Button</Button>
</div>

#### Link
Link buttons are used for navigational actions rather than operations that change data. Also they are great for inline actions.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button variant='link'>Button</Button>
</div>

#### Destructive

Warning buttons are designed to make users think carefully before they use them. They only work if used very sparingly. Most services should not need one.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button variant='destructive'>Button</Button>
</div>

### With Icon

### Icon Only

Icon buuons are great for Actions that need to repeated multiple times across a page. Example: Call a Candidate, Whatsapp Candidate, Reject Candidate, etc.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button variant='outline' size='icon' className='border-gray-200'>
    <PlusIcon className='w-4 h-4' />
  </Button>
  </div>
#### Icon Left
Buttons with icons on the left are great for actions that are special and need to be highlighted.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button variant='outline' className='border-gray-200'>
    
    <PlusIcon className='w-4 h-4' /> <div> Button</div>
  </Button>
</div>

#### Icon Right
Buttons with icons on the right are great for actions that navigate the user to a new page.

<div className='flex  justify-center items-center h-[200px] w-full border border-gray-200 rounded-md mt-3'>
  <Button variant='outline' className='border-gray-200'>
    
   <div> Button</div> <PlusIcon className='w-4 h-4' />
  </Button>
</div>

#### Spacing

- Maintain at least 8px spacing between buttons when they appear in a group.
- For touch interfaces, ensure buttons have at least 44px height for optimal touchability.

#### Text

- Use concise, action-oriented labels (1-3 words when possible).
- Start with a verb to clearly communicate the action.
- Be consistent with capitalization (we use sentence case).

#### Accessibility

- Ensure sufficient color contrast (at least 4.5:1) between text and background.
- Don't rely solely on color to communicate button state or importance.
- Size buttons appropriately for touch targets (minimum 44px Ã— 44px).
