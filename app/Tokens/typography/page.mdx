# Typography

Typography is our system of fonts and text styles. It enhances communication, reinforces our brand identity, and guides users through our digital experiences.

## Overview

Our typography system is designed to create visual hierarchy, improve readability, and maintain consistency across all JobHai products. We use a combination of carefully selected font sizes, weights, and line heights organized into a structured token system.

## Usage Guidelines

### When to use each text style

#### Display and Titles (0-3)
Use for page headers and major section dividers. These create strong visual hierarchy and should be used sparingly to maintain their impact.

#### Titles (4-6)
Use for component headers, subsections, and UI grouping elements. These maintain hierarchy while being more compact.

#### Fluid Body
Use for primary content that needs to adapt between desktop and mobile contexts. The fluid scaling ensures readability across devices.

#### Emphasis
Use to highlight important information without changing the document hierarchy. Perfect for interactive elements, status indicators, and critical information.

#### Subtitles
Use to provide additional context to headings or to label secondary content. These work well for form labels, descriptive text, and metadata.

#### Body and Caption
Use for standard paragraphs, descriptions, and supporting text. Captions are ideal for fine print, metadata, and supplementary information.

### Responsive behavior

Our typography system is responsive by default:
- Mobile tokens use more compact sizes suitable for smaller screens
- Desktop tokens provide more generous sizing for larger displays
- Fluid text elements automatically adjust between contexts

### Best Practices

- **Maintain hierarchy**: Use text styles in a logical progression to create clear visual hierarchy
- **Be consistent**: Use the same text styles for similar elements across your interface
- **Prioritize readability**: Ensure sufficient contrast and appropriate line height for all text
- **Consider context**: Choose text styles appropriate for the amount of content and available space

### Accessibility Considerations

- Maintain WCAG 2.1 AA contrast ratios (4.5:1 for normal text, 3:1 for large text)
- Use proper heading structure (H1 → H2 → H3) for screen reader navigation
- Avoid using text smaller than 12px (text-xs) for essential content
- Ensure text can be scaled up to 200% without loss of content or functionality
