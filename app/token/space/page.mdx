# Spacing

Our design system uses a consistent spacing scale to maintain harmony across the interface. The spacing values are proportional, ensuring visual rhythm and consistent component relationships throughout the application.

## Default Spacing Scale

One spacing unit in our system equals `0.25rem` (4px by default). Our spacing scale follows a progressive pattern that provides appropriate options for various layout needs.

## Usage Guidelines

- **Micro Spacing (0-2)**: Use for tight spacing within components, such as icon-to-text spacing, or internal padding in small components
- **Small Spacing (3-4)**: Appropriate for related element grouping and padding within medium-sized components
- **Medium Spacing (5-8)**: Ideal for separation between components or section padding
- **Large Spacing (10-16)**: Best for major section divisions and page-level spacing
- **Extra Large Spacing (20-64)**: Reserved for dramatic separations, hero sections, and layout-level spacing

## Application

The spacing scale applies to the following CSS properties:
- Padding
- Margin
- Width
- Height
- Gap (for grid and flexbox layouts)
- Translate (for transforms)
- Top/Right/Bottom/Left positioning
